1. üåê Apa itu HTML?

HTML (HyperText Markup Language) adalah bahasa markup yang digunakan untuk membuat struktur dan konten halaman web. HTML bukanlah bahasa pemrograman, melainkan bahasa markup yang menggunakan tag-tag untuk menandai berbagai elemen dalam dokumen.

Analogi Sederhana:

Bayangkan HTML seperti kerangka rumah:

  • üèóÔ∏è Fondasi = struktur dasar HTML
  • üö™ Pintu & Jendela = elemen interaktif (link, button)
  • üñºÔ∏è Dekorasi = CSS (akan dipelajari nanti)
  • ‚ö° Listrik & Air = JavaScript (akan dipelajari nanti)

Contoh HTML Sederhana:

<h1>Selamat Datang di Website Saya</h1> <p>Ini adalah paragraf pertama saya.</p>

Hasil di browser:

Selamat Datang di Website Saya

Ini adalah paragraf pertama saya.

2. üèóÔ∏è Struktur Dasar HTML5

Setiap dokumen HTML5 harus memiliki struktur dasar yang konsisten:

<!-- Deklarasi dokumen HTML5 --> <!DOCTYPE html> <!-- Elemen root/utama --> <html lang="id"> <!-- Bagian HEAD - tidak terlihat di browser --> <head> <!-- Encoding karakter --> <meta charset="UTF-8"> <!-- Viewport untuk responsive design --> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <!-- Judul halaman (muncul di tab browser) --> <title>Judul Halaman</title> </head> <!-- Bagian BODY - konten yang terlihat --> <body> <!-- Konten website di sini --> <h1>Hello World!</h1> </body> </html>

üìù Penjelasan Setiap Bagian:

1. DOCTYPE Declaration

<!DOCTYPE html>
  • Memberitahu browser bahwa ini adalah dokumen HTML5
  • Harus selalu ada di baris pertama
  • Tidak case-sensitive, tapi konvensi menggunakan lowercase

2. HTML Element

<html lang="id">
  • lang="id" menandakan bahasa Indonesia
  • Membantu screen reader dan SEO
  • Contoh lain: lang="en" untuk bahasa Inggris

3. Head Element

Berisi metadata yang tidak terlihat di halaman web:

  • charset="UTF-8": Encoding untuk karakter internasional
  • viewport: Pengaturan untuk responsive design
  • title: Judul yang muncul di tab browser

4. Body Element

Berisi semua konten yang terlihat di browser

Tips: Selalu gunakan indentasi (spasi/tab) untuk membuat kode lebih mudah dibaca!

3. üè∑Ô∏è Memahami Tag vs Element

Apa Perbedaannya?

<!-- TAG PEMBUKA --> <!-- TAG PENUTUP --> <h1> </h1> ‚Üë ‚Üë ‚îî‚îÄ‚îÄ ELEMENT LENGKAP ‚îÄ‚îÄ‚îÄ‚îÄ‚îò <h1>Ini adalah Element</h1> ‚Üë ‚Üë ‚îî‚îÄ‚îÄ CONTENT ‚îÄ‚îÄ‚îò
  • TAG: Pembuka <h1> dan penutup </h1>
  • ELEMENT: Tag + content = <h1>Ini adalah Element</h1>
  • CONTENT: Isi di antara tag pembuka dan penutup

Jenis-Jenis Tag:

1. Container Tags (Berpasangan)

<h1>Heading 1</h1> <p>Paragraf</p> <div>Division</div>

2. Self-Closing Tags (Tidak berpasangan)

<br> <!-- Line break --> <hr> <!-- Horizontal rule --> <img> <!-- Image --> <input> <!-- Form input -->
Perhatian: Dalam HTML5, self-closing tags tidak perlu slash penutup. Jadi <br/> dan <br> sama saja, tapi <br> lebih direkomendasikan.

4. üèóÔ∏è Semantic HTML Elements

Semantic HTML adalah penggunaan elemen HTML yang memberikan makna pada struktur dan konten, bukan hanya tampilan.

Mengapa Semantic HTML Penting?

  • üîç SEO: Search engine lebih mudah memahami konten
  • ‚ôø Accessibility: Screen reader dapat navigasi dengan baik
  • üßë‚Äçüíª Maintainability: Kode lebih mudah dibaca dan dipelihara
  • üì± Future-proof: Kompatibel dengan teknologi masa depan

Semantic Elements HTML5:

Struktur Semantic Layout:

<header> <!-- Header website/halaman --> <h1>Nama Website</h1> <nav> <!-- Navigasi utama --> <ul> <li><a href="#">Home</a></li> <li><a href="#">About</a></li> </ul> </nav> </header> <main> <!-- Konten utama halaman --> <article> <!-- Artikel independen --> <h2>Judul Artikel</h2> <section> <!-- Bagian dalam artikel --> <h3>Sub Judul</h3> <p>Konten artikel...</p> </section> </article> <aside> <!-- Sidebar/konten sampingan --> <h3>Artikel Terkait</h3> </aside> </main> <footer> <!-- Footer website --> <p>© 2025 Nama Website</p> </footer>

üìã Daftar Semantic Elements:

  • <header>: Header halaman/section
  • <nav>: Navigasi menu
  • <main>: Konten utama (hanya 1 per halaman)
  • <article>: Konten independen (blog post, news article)
  • <section>: Bagian terkait dalam konten
  • <aside>: Konten sampingan (sidebar, related links)
  • <footer>: Footer halaman/section
  • <figure>: Gambar dengan caption
  • <figcaption>: Caption untuk figure
Tips: Gunakan semantic elements daripada <div> generik ketika memungkinkan!

5. ‚úÖ HTML Best Practices

1. Proper Nesting (Nested dengan Benar)

‚ùå Salah:

<p><strong>Bold text</p></strong> <!-- Tag tidak sejajar -->

‚úÖ Benar:

<p><strong>Bold text</strong></p> <!-- Tag sejajar -->

2. Consistent Indentation

‚úÖ Gunakan 2 atau 4 spasi konsisten:

<div> <h1>Judul</h1> <p> Ini adalah paragraf dengan <strong>text bold</strong> di dalamnya. </p> </div>

3. Use Lowercase Tags

‚ùå Hindari:

<DIV><H1>TITLE</H1></DIV>

‚úÖ Gunakan:

<div><h1>Title</h1></div>

4. Always Quote Attributes

‚ùå Tidak konsisten:

<div class=container id=main>

‚úÖ Selalu gunakan quotes:

<div class="container" id="main">

5. Provide Alt Text for Images

<!-- Selalu sediakan alt text untuk accessibility --> <img src="logo.jpg" alt="Logo perusahaan ABC">
Ingat: HTML yang valid dan semantic akan membuat website Anda lebih profesional dan mudah diakses oleh semua pengguna!
